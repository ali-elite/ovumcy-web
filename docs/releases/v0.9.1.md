# Ovumcy v0.9.1

Release date: 2026-04-15

## Highlights

- Public patch release that supersedes `v0.9.0`.
- Tightens the browser recovery-code confirmation flow so DOM-provided continue paths can no longer steer navigation outside the expected Ovumcy app routes.
- Cleans up the last local Go formatting and cyclomatic-complexity warnings in the touched password-change, recovery, migration-bootstrap, cycle-hero, and TLS-helper code paths without changing runtime behavior.

## Security and quality

- Recovery-code confirmation now allows only the expected same-origin app routes (`/dashboard`, `/onboarding`, `/settings`) when the client transitions after a confirmed recovery-code acknowledgement.
- Browser regression coverage now includes a recovery-flow scenario that injects a hostile continue path and proves the UI still lands on the safe app route.
- The maintenance cleanup keeps the existing auth/session, settings, and prediction contracts intact while making the surrounding test and helper code easier to review.
- The release was validated with `npm run build`, `npm run lint`, focused recovery browser e2e, `gofmt -w -s` on the touched Go files, and `go test ./internal/api ./internal/db ./internal/services ./scripts/e2e-tls-cert`.

## Upgrade notes

- No new database migration is introduced in this patch release.
- Tagged images publish under `ghcr.io/ovumcy/ovumcy-web:v0.9.1`.
- Existing deployments can upgrade in place and pin `OVUMCY_IMAGE=ghcr.io/ovumcy/ovumcy-web:v0.9.1` if they do not want to track future tags manually.
