# Ovumcy v0.8.0

Release date: 2026-03-29

## Highlights

- Added optional OpenID Connect sign-in for self-hosted deployments, including `hybrid` and `oidc_only` login modes.
- Added guarded owner auto-provision, stored `(issuer, subject)` account links, and local-password enablement for OIDC-only accounts.
- Added provider logout support through a same-origin bridge without weakening the hardened HTML callback model.

## Security and quality

- Provider logout state now stays server-side and is keyed by the auth-session `sid`, which avoids oversized auth headers and keeps raw provider end-session parameters out of long-lived cookies.
- `OIDC_CA_FILE` and `SECRET_KEY_FILE` now share the same bounded regular-file validation contract for operator-managed trust material.
- The release was validated with `go test ./...`, `staticcheck ./...`, `gosec ./cmd/... ./internal/...`, `govulncheck ./...`, `npm audit --json`, `npm run lint`, `npm run build`, cross-browser auth/recovery smoke, and all three OIDC browser lanes on Chromium, Firefox, and WebKit.

## Upgrade notes

- This release includes forward-only database migrations. Back up your SQLite file or Postgres data before upgrading, then restart Ovumcy and let the built-in migration runner apply schema version `016_oidc_logout_states`.
- Tagged images publish under `ghcr.io/ovumcy/ovumcy-web:v0.8.0`.
- Existing deployments can upgrade in place and pin `OVUMCY_IMAGE=ghcr.io/ovumcy/ovumcy-web:v0.8.0` if they do not want to track future tags manually.
- OIDC provider compatibility remains intentionally strict: Keycloak, authentik, and Authelia are supported under the hardened browser model; Dex and Pocket ID remain excluded because they require query-string callback parameters.
